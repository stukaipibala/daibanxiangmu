# 每日待办 (Daily Todo) - 产品需求文档 (PRD)

---

## 1. 产品概述

### 1.1 产品名称
每日待办 (Daily Todo)

### 1.2 产品定位
一款极简、轻量的每日待办事项记录工具，专注于"快速记录"和"历史回顾"两个核心场景，让用户用最少的操作完成每日任务管理。

### 1.3 目标用户
- 需要简单记录每日工作/生活事项的普通用户
- 不喜欢复杂功能，追求效率的轻量用户
- 希望回顾每日完成情况的用户

### 1.4 核心价值主张
> **"3秒记录，一键回顾"** —— 用最简单的方式管理每一天。

---

## 2. 需求背景

### 2.1 用户痛点
- 现有待办应用功能臃肿，学习成本高
- 复杂的分类、标签、优先级让用户望而却步
- 历史记录查看不便，难以回顾每日完成情况
- 打开应用后需要多次点击才能开始记录

### 2.2 解决思路
- 砍掉一切非必要功能，只保留"记录"和"查看"
- 打开应用即进入输入状态，零门槛使用
- 日历式历史回顾，直观查看每日记录

## 3. 功能需求

### 3.1 功能清单

| 功能模块 | 功能点 | 优先级 | 说明 |
|---------|--------|--------|------|
| 今日待办 | 快速添加待办 | P0 | 输入框+回车，3秒完成记录 |
| 今日待办 | 标记完成/取消完成 | P0 | 点击复选框切换状态 |
| 今日待办 | 删除待办 | P0 | 左滑或点击删除按钮 |
| 今日待办 | 显示今日日期 | P1 | 顶部显示"今天 5月15日 周四" |
| 历史记录 | 日历视图 | P0 | 月历形式，有记录的日期高亮 |
| 历史记录 | 日期详情 | P0 | 点击日期查看当日所有待办 |
| 历史记录 | 完成状态展示 | P1 | 详情中显示已完成/未完成 |
| 数据管理 | 本地存储 | P0 | 数据保存在本地，无需登录 |
| 数据管理 | 数据持久化 | P0 | 应用关闭后数据不丢失 |

### 3.2 非功能需求

| 需求项 | 要求 |
|--------|------|
| 启动速度 | 冷启动 < 1秒 |
| 操作响应 | 添加/完成/删除操作 < 200ms |
| 数据安全 | 本地存储，不上传云端 |
| 离线可用 | 完全离线使用，无需网络 |
| 安装包体积 | < 5MB（如为App）|

## 4. 详细功能设计

### 4.1 今日待办页面（首页）

#### 4.1.1 页面布局
```
┌─────────────────────────────┐
│  今天  5月15日  周四          │  ← 顶部日期栏
├─────────────────────────────┤
│  [ 输入待办事项...      ] +  │  ← 输入框+添加按钮
├─────────────────────────────┤
│  ☐ 完成项目周报             │  ← 待办列表
│  ☐ 下午3点开会              │
│  ☑ 买牛奶                   │  ← 已完成（划线）
│  ☐ 晚上跑步30分钟           │
│                             │
│  ── 已完成 1 项 ──          │  ← 已完成区域（可折叠）
│  ☑ 回复客户邮件             │
├─────────────────────────────┤
│  [今日]  [历史]  [设置]     │  ← 底部导航栏
└─────────────────────────────┘
```

#### 4.1.2 交互说明

**添加待办**
- 点击输入框 → 弹出键盘 → 输入内容 → 点击"+"或按回车 → 添加到列表顶部
- 输入框支持多行文本（换行自动扩展高度）
- 空内容不允许提交

**标记完成**
- 点击待办项左侧复选框 → 该项移至"已完成"区域
- 再次点击 → 移回"待完成"区域
- 完成项文字显示删除线，颜色变灰

**删除待办**
- 左滑待办项 → 显示"删除"按钮 → 点击删除
- 删除时有轻微动画（向左滑出）
- 无撤销功能（保持极简）

**已完成区域**
- 默认展开，显示所有已完成项
- 点击区域标题可折叠/展开
- 显示已完成数量

#### 4.1.3 空状态
- 今日无任何待办时，页面中央显示：
  - 图标：一个空白的清单图标
  - 文字："今天还没有待办事项"
  - 提示："在上方输入框添加你的第一个待办吧"

#### 4.2.1 页面布局
```
┌─────────────────────────────┐

├─────────────────────────────┤
│  日  一  二  三  四  五  六  │  ← 星期标题
│  28  29  30   1   2   3   4 │

│  12  13  14 [15]  16  17  18│  ← 今天高亮框
│  19  20  21  22  23  24  25 │     有记录日期下方有小圆点
│  26  27  28  29  30  31     │
├─────────────────────────────┤
│  5月15日 周四               │  ← 选中日期标题
├─────────────────────────────┤
│  ☑ 完成项目周报             │  ← 当日待办列表
│  ☑ 下午3点开会              │     显示完成状态
│  ☐ 买牛奶                   │
│  ☑ 晚上跑步30分钟           │
│  ☐ 回复客户邮件             │
├─────────────────────────────┤
│  [今日]  [历史]  [设置]     │  ← 底部导航栏
└─────────────────────────────┘
```

#### 4.2.2 交互说明

**月份切换**
- 点击左右箭头切换上/下月
- 支持左右滑动手势切换月份
- 月份标题点击可快速选择年份月份（弹出选择器）

**日期选择**
- 点击日期 → 下方显示该日待办列表
- 有记录的日期下方显示小圆点（已完成全部显示绿色，有未完成显示橙色）
- 今天日期用特殊边框高亮
- 选中日期用背景色高亮

**日期详情**
- 显示当日所有待办及完成状态
- 只读模式，不可编辑（保持历史记录纯粹性）
- 空记录显示："这一天没有记录待办事项"

## 5. 数据模型

### 5.1 待办事项 (Todo)
```json
{
  "id": "uuid-string",
  "content": "待办内容文本",
  "completed": false,
  "createdAt": "2026-05-15T10:30:00Z",
  "completedAt": null
}
```

### 5.2 存储结构
```json
{
  "todos": {
    "2026-05-15": [
      { "id": "1", "content": "完成项目周报", "completed": true, ... },
      { "id": "2", "content": "买牛奶", "completed": false, ... }
    ],
    "2026-05-14": [
      { "id": "3", "content": "整理桌面", "completed": true, ... }
    ]
  }
}
```
- Key：日期字符串（YYYY-MM-DD）
- Value：当日待办数组
- 存储方式：LocalStorage（Web）/ SQLite（App）/ UserDefaults（iOS）

## 7. 交互与动画

### 7.1 页面切换
- 底部导航切换：淡入淡出，时长 200ms
- 日历月份切换：左右滑动，时长 300ms

### 7.2 列表操作
- 添加项：从顶部滑入，时长 200ms
- 删除项：向左滑出，时长 200ms
- 标记完成：复选框缩放动画（scale 1→1.2→1），时长 150ms
- 项移动：完成/取消完成时，平滑移动至对应区域，时长 300ms

### 7.3 反馈
- 添加成功：轻微震动（移动端）
- 删除成功：无额外反馈（动画已足够）
- 空内容提交：输入框轻微抖动

## 8. 异常处理

| 场景 | 处理方式 |
|------|----------|
| 存储空间不足 | 提示"存储空间不足，请清理后重试" |
| 数据读取失败 | 显示空状态，不崩溃 |
| 输入超长文本（>500字）| 限制输入长度，超出不可输入 |
| 跨天使用 | 自动切换到新的一天，历史数据保留 |

## 11. 附录

### 11.1 竞品参考
- **Microsoft To Do**：功能过多，不够轻量
- **Google Tasks**：相对简洁，但历史查看不便
- **极简待办**：接近目标，但缺少日历式历史回顾

### 11.2 设计原则
1. **减法原则**：每个功能加入前问"用户真的需要吗？"
2. **零学习成本**：新用户打开即用，无需教程
3. **秒级操作**：任何操作不超过3次点击
4. **视觉降噪**：减少装饰元素，内容即界面
